# MedBook — безопасность и исправления

Ниже кратко описаны основные риски, которые были учтены в проекте, и текущие меры защиты.

## 1. SQL Injection

### Риск

Если подставлять пользовательский ввод в SQL как строку, можно получить доступ к лишним данным или повредить БД.

### Что сделано

- все основные запросы к SQLite используют параметры `?`
- фильтрация врачей, получение слотов, авторизация и работа с записями идут через параметризованные запросы

## 2. XSS

### Риск

Если небезопасно выводить пользовательские данные в HTML, можно выполнить чужой JavaScript в браузере.

### Что сделано

- UI построен на React, который экранирует текст по умолчанию
- сервер запрещает опасные символы в имени пользователя
- включён CSP через `helmet`

## 3. CSRF

### Риск

Refresh cookie может быть использована с другого сайта, если не ограничить её поведение.

### Что сделано

- refresh token хранится в `HttpOnly` cookie
- для cookie включён `SameSite=strict`
- в production cookie работает как `secure`
- CORS ограничен списком разрешённых origin

## 4. Brute Force и abuse

### Риск

Без ограничения частоты запросов можно быстро перебирать пароли или перегружать auth-эндпоинты.

### Что сделано

- на `/api/auth/*` включён `express-rate-limit`
- JSON body ограничен до `10kb`
- включён `hpp` против HTTP parameter pollution

## 5. Гонки при записи на слот

### Риск

Два пользователя могут почти одновременно отправить запись на одно и то же время.

### Что сделано

- запись на слот выполняется внутри транзакции `BEGIN IMMEDIATE`
- слот бронируется только через `UPDATE ... WHERE is_booked = 0`
- на `appointments(slot_id)` создан уникальный индекс

## 6. Дополнительные исправления устойчивости

- async-маршруты обёрнуты в единый `asyncHandler`, чтобы ошибки не терялись
- исправлена работа `sitemap.xml`, которая раньше использовала несовместимый API БД
- добавлена автозагрузка демо-данных для пустой БД в development-режиме
- слоты и напоминания теперь фильтруются по локальному времени, без показа прошедших записей
- просроченные записи автоматически переводятся в `completed`
